-WriteToBeepconAndReset:NewiBeaconTime:NewBeepconTime:NewEddystoneUIDTime:NewEddystoneURLTime:NewEddystoneTLMTime:NewConfigTime:

This will set the new Tx Pattern to the BEEPCON. A pattern means how many seconds the BEEPCON DEVICE is broadcasting for each supported format. (iBeacon, Beepcon, Eddystone UID, Eddsytone URL and Eddystone TLM). The broadcast is sequential. Also you decide how many time from booting the BEEPCON DEVICE you can access the programming parameters. (As a BEEPCON in case beepcontime is zeroed or as a Eddystone URL-config). All time units are in seconds.

-WriteToBeepconAndReset:NewPassword:

This will Set the a new Password for the BEEPCON.

-WriteToBeepconAndReset:NewTxDbCalibrationData:

This will Set the NewCalData as the new Calibrated TXPower for the actual TxPower Value.

-WriteToBeepconAndReset:NewTxPower:

This will set the new TxPower for the BEEPCON. With this value you can set the radiofrecuency power level that the BEEPCON will use to transmit its data, thus having some control of the range of the BEEPCON.

-WriteToBeepconAndReset:NewTxRate:

This will set the new TxRate for the BEEPCON. With this value you can set the rate at wich the BEEPCON will broadcast its data (broadcasts per second). The Rate affects directly the power comsumption of the BEEPCON so this parameter must be changed with care, as affects the battery life. The minimun TxRate that fits your use case is the recommended value.

-WriteToBeepconAndReset:NewUUID:

This will set the new UUID for the BEEPCON. Be aware that once changed the UUID, in order to find again the BEEPCON you need to set this UUID via the setUUIDToScan: method prior to start a Scan.

-WriteToBeepconAndReset:QuietMode:

This will set the the BEEPCON to quiet mode. This means that the Beeping function will be limited to short "beeps" and this will override the user beeping preferences.

-WriteToBeepconAndResetDefaultFactoryCalibrationData:

This will Reset the calibration TxPower for the actual TxPower value to factory settings.

CheckTheConnectedBeepconFirmwareCompatibilityWithOtaName:error:

This functions is used to check if a firmware file is compatible with the connected device.

- (BOOL) 
        CheckTheConnectedBeepconFirmwareCompatibilityWithOtaName:(NSString *)firmwarename 
        error:(NSError **)error;
Parameters
firmwarename

The complete name of the file from where the firmware data was extracted used to check compatibility Ex. VC_M001H0300S0104010128F0007.ota

error

Reference to an error to return a friendly description of non compatibilities.

Return Value

TRUE if Compatible, FALSE otherwise.

Discussion

The process to check the compatibility is

  1. connect to the BEEPCON device (with password)

  2. Once password is validated make a call to ReadVersionNumber: and wait its corresponding answer Read VersionNumber will refresh the BEEPCON device actual version code numbers, that are internally needed to perform this check.

  3. Once version is readed you can call this function to check compatibility of your files.

Return Value

Boolean TRUE if scan restarted

During very Long SCANS (more than for example 20 segs) it seems that IOS becomes LAZY giving reports of new packets. Calling this function will refresh the SCAN order to IOS and avoids this slow down. The delegate ITBeepconsManagerDidStartScanning: will be called. This function will not reset the information stored for the Original Scan.

This will set the new TxPower for the BEEPCON. With this value you can set the radiofrecuency power level that the BEEPCON will use to transmit its data, thus having some control of the range of the BEEPCON.

- (void) WriteToBeepconAndReset:(ITFoundedBeepcon *)WithThisFoundedBeepcon 
        NewTxPower:(unsigned char)txpower;
Parameters
WithThisFoundedBeepcon

The ITFoundedBeepcon Object to take the action

txpower

The new tx power value (0..MaxValueForTxPower)

Discussion

WARNING:

Needs connection with password validated.

WARNING:

The action will Autoreset the BEEPCON and Disconnect the current connection.

WARNING:

the valid values can depend on BEEPCON Model, once connected a password validated you can read the valid values from MaxValueForTxPower readonly properties.

See

ITBeepconsManagerDidWriteToBeepconNewTxPower:withError:

WriteToBeepconAndReset:NewTxRate:

This will set the new TxRate for the BEEPCON. With this value you can set the rate at wich the BEEPCON will broadcast its data (broadcasts per second). The Rate affects directly the power comsumption of the BEEPCON so this parameter must be changed with care, as affects the battery life. The minimun TxRate that fits your use case is the recommended value.

WriteToBeepconAndReset:NewUUID:

This will set the new UUID for the BEEPCON. Be aware that once changed the UUID, in order to find again the BEEPCON you need to set this UUID via the setUUIDToScan: method prior to start a Scan.

- (void) WriteToBeepconAndReset:(ITFoundedBeepcon *)WithThisFoundedBeepcon 
        NewUUID:(CBUUID *)NewUUID;
Parameters
WithThisFoundedBeepcon

The ITFoundedBeepcon Object to take the action

NewUUID

The new 16 bytes UUID

Discussion

WARNING:

Needs connection with password validated.

WARNING:

The action will Autoreset the BEEPCON and Disconnect the current connection.

WARNING:

changing the UUID affect both the BEEPCON broadcast the iBeacon Broadcast, so you need to change also the UUID to Locate in ITBeepconsLocator if you want this BEEPCON to be detected as iBeacon.

This will set the the BEEPCON to quiet mode. This means that the Beeping function will be limited to short "beeps" and this will override the user beeping preferences.

Discussion

WARNING:

Due that this property is updated when CoreBluetooth informs the SDK about the actual state, it´s value is not updated inmediately once this class is instantiated. This means for example, that if you instatiate the class and inmediately check this status(For example to start a Scan), you can get a FALSE value. You can use the ITBeepconsManagerHasChangedAuthorizationStatus: to wait until iOS updates the value, with some extra own logic to perform this kind of tasks...

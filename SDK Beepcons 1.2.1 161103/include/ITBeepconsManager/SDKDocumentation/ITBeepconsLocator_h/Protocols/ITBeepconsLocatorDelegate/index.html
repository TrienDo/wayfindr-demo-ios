ITBeepconsLocatorDelegate

Extends Protocol:
NSObject
Declared In:
ITBeepconsLocator.h

Introduction

Protocol to be implemented by the ITBeepconsLocator owner class to receive events via the delegate pattern

Discussion

The class that instantiates a ITBeepconsLocator object needs to implement this delegate to receive all the required answers fired by ITBeepconsLocator. At the moment of this release all the delegate methods are declared as Required.

ITBeepconsLocatorDidStartLocating:withRegionIdentifier:

Called to indicate that the ITBeepconsLocator started locating BEEPCONS

@required

- (void) ITBeepconsLocatorDidStartLocating:(id)thislocator 
        withRegionIdentifier:(NSString *)identifier;
Parameters
thislocator

the instance of the class ITBeepconsLocator answering via this protocol method

identifier

the identifier of the region that started locating (Ex. "com.ilunion.MyGroupOfBeepconsOneForLocate"). This identifier is supplied by the client app when calling StartLocatingBeepconsWithHiId:LowId:Identifier:

Discussion

Once the client app calls the StartLocatingBeepconsWithId:LowId:Identifier function, this function is called via delegate to confirm the client app that the IOS system is now monitoring for BEEPCONS. Monitoring for BEEPCONS is not the same as scanning for BEEPCONS on ITBeepconsManager. Monitoring is used to inform the app that the user has entered or exit a region with BEEPCONS and this work is automatically done by IOS.

See

StartLocatingBeepconsWithHiId:LowId:Identifier:

ITBeepconsLocatorReportingBeepconAsIbeacon:withRegionIdentifier:major:minor:rssi:accuracy:proximity:

Called to report that a BEEPCON has been encountered under the iBeacon Format

@required

- (void) ITBeepconsLocatorReportingBeepconAsIbeacon:(id)thislocator 
        withRegionIdentifier:(NSString *)identifier major:(unsigned int)hiident 
        minor:(unsigned int)loident rssi:(int) rssi accuracy:(double) Accuracy 
        proximity:(int) proximity;
Parameters
thislocator

the instance of the class ITBeepconsLocator answering via this protocol method

identifier

the identifier of the region that started reporting (Ex. "com.ilunion.MyGroupOfBeepconsOneForReporting"). This identifier is supplied by the client app when calling StartReportingBeepconstartWithHiId:LowId:WithRegionIdentifier:

hiident

the High value of the identifier of this Beepcon (also known as Major in Apple Docs)

loident

the Low value of the identifier of this Beepcon (also known as Major in Apple Docs)

Accuracy

estimated accuracy

proximity

a estimad value indicating the proximity to the Beepcon

WARNING:

For a more explained description of this parameters please refer to the apple CoreLocation framework documentation. See the CLBeaconRegion Class.

WARNING:

This parameters are redirected from CoreLocatioin Framework to the client App with no more processing task inside this SDK.

Discussion

when the app request to IOS to report BEEPCONS (via StartReportingBeepconsWithId:LowId:Identifier) as soon as it encounters them , the SDK calls this delegate function to inform the client app that BEEPCON data compatible with the iBeacon Profile has been received.
